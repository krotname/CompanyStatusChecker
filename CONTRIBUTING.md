## Contributing

Please keep changes small and aligned with the project architecture:

- Validate INN-related logic in `com.krotname.checker.validation`.
- Keep Dadata parsing and integration in `com.krotname.checker.client`.
- Add/adjust tests for behavioral changes (unit + integration).
- Do not add secrets to the repo.
- Update `README.md` when user-facing behavior changes.

### Commit style

- Use imperative commit messages.
- Mention test commands and CI impact in PR descriptions.
